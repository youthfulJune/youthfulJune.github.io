一、栈
1.1 栈的概念
    栈(stack),也称为堆栈，是一种容器，可存入数据元素、访问元素、
    删除元素，其特点在于只能允许在容器的一端进行加入数据和输出数
    据的运算。没有位置概念，保证任何时候可以访问、删除的元素都是
    此前最后存入的那个元素，确定了一种默认的访问顺序。

1.2 栈的运作原理
    由于栈数据结构只允许在一端进行操作，故按照后进先出
    (LIFO,Last In First Out)的原理运作


1.3 栈结构的实现
    栈可以用顺序表实现，也可以用链表实现。

    1.3.1 用Python的列表list实现栈
        class StackUnderflow(ValueError):
            '''与ValueError所有的异常有所区分'''
            pass
        class ListStack(object):
            '''list实现的栈'''
            def __init__(self):
                self._elem = []

            def push(self,item):
                '''添加一个新的元素item到栈顶'''
                self._elem.append(item)

            def pop(self):
                '''弹出栈顶元素'''
                if self._elem == []:
                    raise  StackUnderflow("lickStack.pop()")
                return self._elem.pop()

            def peek(self):
                '''返回栈顶元素'''
                if self._elem == []:
                    raise StackUnderflow("ListStack.peek()")
                return self._elem[len(self._elem)-1]#self._elem(-1)

            def is_empty(self):
                '''判断栈是否为空'''
                return self._elem == []

            def size(self):
                '''返回栈的元素个数'''
                return len(self._elem)

    1.3.2  用链表的方式实现栈
            class StackUnderflow(ValueError):
                pass

            class Node(object):
                def __init__(self,item,next=None):
                    self.item = item
                    self.next = next

            class LinklistStack(object):
                '''链表实现的栈'''

                def __init__(self):
                    self._top = None

                def push(self,item):
                    '''添加一个新的元素item到栈顶'''
                    self._top = Node(item,self._top)

                def pop(self):
                    '''弹出栈顶元素'''
                    if self._top == []:
                        raise  StackUnderflow("lickStack.pop()")
                    #弹出栈元素后，要将self,_top转换位置
                    cur = self._top
                    self._top = cur.next
                    return cur.item

                def peek(self):
                    '''返回栈顶元素'''
                    if self._top == []:
                        raise StackUnderflow("ListStack.peek()")
                    return self._top.item

                def is_empty(self):
                    '''判断栈是否为空'''
                    return self._top == []

                def size(self):
                    '''返回栈的元素个数'''
                    count = 0
                    cur = self._top
                    while cur != None:
                        count += 1
                        cur = cur.next
                    return count

二、队列
2.1  队列的概念
    队列(queue)是只允许在一端进行插入操作，而在另一端进行删除操
    作的线性表
2.2  队列的运作原理
    队列是一种先进先出的(First In First Ou)）的线性表,简称FIFO。
    允许插入的一端为队尾，允许删除的一端为队头。队列不允许在中间
    部位进行操作！假设队列是q=（a1，a2，……，an），那么a1就是队头
    元素，而an是队尾元素。这样我们就可以删除时，总是从a1开始，而
    插入时，总是在队列最后。这也比较符合我们通常生活中的习惯，排
    在第一个的优先出列，最后来的当然排在队伍最后。

2.3  队列结构实现
     队列也可以用顺序表实现，也可以用链表实现。

    2.3.1  用Python的列表list实现队列

线性表之链式存储

发表于 2017-05-08 | 分类于 python | 阅读次数
一、线性表的概念
线性表是最基本、最简单、也是最常用的一种数据结构，是某类元素的一个集合，用元素在序列里的位置
和顺序来表示实际应用中的某种有意义的信息，或者表示数据之间的某种关系的元素组。
根据线性表的实际存储方式，分为两种实现模型：
    1.顺序表--将元素顺序的存放在一块连续的存储区里，元素间的顺序关系有它们的存储顺序自然
        表示
    2.链表--将元素存放在通过链接构造起来的一系列存储块中
二、链表
链表是一种常见的基础数据结构，不像顺序表一样连续存储数据，而是在每一个节点(数据存储单元)里
存放下一个节点的位置信息(即地址)；从而解决了顺序表在进行扩充时需要进行数据搬迁的问题
2.1 单链表
单向链表也叫单链表，是链表中最简单的一种形式，它的每个节点包含两个域，一个信息域（元素
域）和一个链接域。这个链接指向链表中的下一个节点，而最后一个节点的链接域则指向一个空值。

                cur = cur.next
            cur.next = node
            node.next = self.header

            self.append(item)

            return

                cur = cur.next

        if cur.elem == item:
            if cur == self.header:
                self.header = None
            else:
                forword_cur.next = self.header

        '''查找节点是否存在'''
        cur = self.header
        if cur == None:
            print("没有相关节点信息...")
        elif cur.elem == item:
            print("已找到节点%s" % item)

        while cur.next != self.header:
            cur = cur.next
            if cur.elem == item:
                print("已找到节点%s" % item)

python全局解释器锁--GIL

发表于 2017-05-02 | 分类于 python | 阅读次数
一、GIL产生的背景
1.1、GIL全称是Global Interpreter Lock(全局解释器锁)
1.2、Python被设计之初，都是单核处理器,要知道每个CPU在同一个时间只能执行一个线程，
    故在单核CPU下的多线程都只能是并发，在同一时刻只有一个线程被执行，就不会出现
    全局变量数据安全的问题；
1.3、多核处理器出现后，多个线程在同一时刻被执行，然而对于多线程共享的全局变量来说，
    就会出现数据安全的问题
1.4、Python开发者为了解决这一弊端，就有了GIL；某个线程想要被执行，就必须先拿到GIL，
    才能进入CPU执行
二、GIL–python多线程
2.1 下面我们针对多线程在不同核数的计算机上出现的情况，来充了解GIL实现的功能：
    还是先说一下并发与并行吧，并发和并行从宏观上来讲都是同时处理多任务的概念，但又有
    区别：并发是指多个事件依据操作系统的算法在同一时间间隔内发生;并行是指多个事件在
    同一时刻发生。

下面的代码实现了多线程，主线程和3个子线程均为死循环：
    import threading
    def test():

肯定是有用的：对于**IO密集型代码**(文件处理、网络爬虫),多线程能够有效的提升效率
            (单线程下有IO操作会出现IO等待，造成不必要的时间浪费，而开启多线程能在
            线程A等待时，自动切换到线程B，可以不浪费CPU的资源，从而能提升程序执行效率)。
            但对于**CPU密集型代码**就不那么友好了，CPU密集型代码由于计算工作多，计数器
            很快达到阈值，然后触发GIL的释放与再竞争,多个线程来回切换消耗系统资源
三、Python中充分利用多核CPU
3.1 多进程
在Python的多进程下，每个进程都有各自独立的GIL，互不干扰(不会像多线程那样共享GIL)，这样就可以真正意义上的并行执行，故在Python中，在多核CPU的充分利用上，多进程优于多线程
下面的代码实现了多进程：1个主进程和1个子进程
from multiprocessing import Process
def test():
    while True:
     pass
p1 = Process(target=test)
p1.start()
while True:
    pass

    #-*-coding:utf-8-*-
